# Contributing

Thanks for helping improve Print Status Bar! A few guidelines:

- Open an issue before big changes/feature work.
- Fork → create a feature branch → push → open a PR.
- Use clear commit messages (one-line summary, optional body).
- Add unit or integration checks where possible (see `.github/workflows/ci.yml`).
- For firmware changes, update `Firmware/docs/flashing.md` and `Firmware/README.md`.

Labeling:
- `bug` for reproducible problems
- `enhancement` for new features
- `help wanted` when maintainers request assistance
